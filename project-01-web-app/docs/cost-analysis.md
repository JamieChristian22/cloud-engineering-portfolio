# Cost Analysis

## Cost Strategy

The project was designed to use low-cost cloud resources where possible.

## Cost Risks

Virtual machines, disks, and public IP addresses can create charges if left running.

## Cost Control Actions

1. Use free-tier or low-cost VM sizes where possible.
2. Stop VMs when not testing.
3. Delete unused disks.
4. Delete unused public IPs.
5. Delete resource groups when finished.
6. Avoid paid load balancers until needed.

## Recommendation

After collecting all screenshots, stop or delete cloud resources to prevent unnecessary charges.
