# Project 01 — Multi-Cloud Web Application Platform

## Executive Summary

This project demonstrates the deployment of a web application across three major cloud providers: Amazon Web Services, Microsoft Azure, and Google Cloud Platform. Each cloud provider hosts an Apache web server running on a virtual machine with public web access, cloud-native network controls, and monitoring visibility.

## Business Problem

Organizations often operate across multiple cloud providers to improve flexibility, reduce vendor lock-in, support different business units, and increase resilience. Cloud engineers must understand how to deploy, secure, monitor, and document workloads across AWS, Azure, and GCP.

## Solution

A static Apache web application was deployed on:

| Cloud Provider | Compute Service | Web Server | Monitoring |
|---|---|---|---|
| AWS | Amazon EC2 | Apache HTTP Server | Amazon CloudWatch |
| Azure | Azure Virtual Machine | Apache HTTP Server | Azure Monitor |
| GCP | Compute Engine | Apache HTTP Server | Google Cloud Monitoring and Logging |
