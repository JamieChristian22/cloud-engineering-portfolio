# Monitoring Design

## Objective

Provide operational visibility across AWS, Azure, and GCP using cloud-native monitoring platforms.

## AWS Monitoring

Amazon CloudWatch was used to review:

- CPU utilization
- Network in
- Network out
- Instance status checks

## Azure Monitoring

Azure Monitor was used to review:

- CPU metrics
- VM health
- Network metrics
- Platform activity

## GCP Monitoring

Google Cloud Monitoring and Cloud Logging were used to review:

- CPU usage
- Network traffic
- Disk activity
- VM logs
- Instance activity

## Observability Outcome

The monitoring design gives cloud engineers visibility into infrastructure health, traffic patterns, and operational events across all three platforms.
