# Runbook - Multi-Cloud Threat Detection Platform

## Objective

Provide repeatable steps to validate and investigate the security controls in this project.

## Steps

1. Confirm cloud environment scope.
2. Verify control configuration in AWS, Azure, and GCP.
3. Trigger a safe test event where applicable.
4. Review logs, alerts, policies, or findings.
5. Capture evidence screenshots.
6. Document findings and remediation.

## Remediation Checklist

- Remove excessive permissions.
- Disable exposed resources.
- Rotate affected secrets or keys.
- Update policies.
- Add alerting or monitoring gaps.
- Document the lesson learned.
