# Security Design

## Objective

Protect the web application infrastructure using cloud-native network and identity controls.

## AWS Security Controls

- Security Group controls inbound access
- SSH restricted where possible
- HTTP/HTTPS configured for application access
- IAM principles documented for future enhancement

## Azure Security Controls

- Network Security Group controls inbound access
- Azure RBAC used for cloud access management
- Managed identity and Azure access controls reviewed during SSH setup

## GCP Security Controls

- Firewall rules control HTTP and HTTPS access
- IAM controls administrative access
- Cloud Logging provides activity visibility

## Security Principles

- Least privilege
- Defense in depth
- Controlled administrative access
- Monitoring and logging
- Network traffic filtering
