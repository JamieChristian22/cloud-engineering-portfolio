# Security Controls

## Overview

This document explains the security controls used across AWS, Azure, and GCP for the multi-cloud web application platform.

## AWS Controls

- Security Group controls inbound traffic.
- SSH access used for administration.
- HTTP allowed for public website access.
- HTTPS documented for future production readiness.
- CloudWatch provides operational visibility.

## Azure Controls

- Network Security Group controls inbound traffic.
- Azure RBAC controls cloud resource access.
- Azure Monitor provides performance visibility.
- Azure Cloud Shell/CLI SSH supports administrative access.

## GCP Controls

- Firewall rules control inbound traffic.
- IAM controls project and VM access.
- Cloud Monitoring provides performance visibility.
- Cloud Logging provides activity visibility.

## Security Outcome

The project demonstrates the ability to apply cloud-native security controls across all three major cloud providers.
