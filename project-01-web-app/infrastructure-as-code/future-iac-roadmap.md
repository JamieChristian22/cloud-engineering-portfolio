# Infrastructure as Code Roadmap

## Purpose

The current version of the project was deployed manually to demonstrate hands-on console, SSH, and Linux administration skills. A future version should automate deployment with Terraform.

## Planned Terraform Components

### AWS

- EC2 instance
- Security Group
- VPC/subnet selection
- CloudWatch alarms

### Azure

- Resource Group
- Virtual Network
- Subnet
- Public IP
- Network Security Group
- Virtual Machine

### GCP

- Compute Engine VM
- Firewall rules
- Network tags
- Monitoring alert policies

## Why Terraform Matters

Terraform makes cloud infrastructure repeatable, version-controlled, auditable, and easier to destroy when finished.
