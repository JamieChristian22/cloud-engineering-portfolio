# Incident Management

## Purpose

Incident management provides a structured process for responding to service issues.

## Incident Workflow

1. Detect issue through monitoring or user report.
2. Confirm impact.
3. Check cloud VM health.
4. Check Apache service status.
5. Check network rules.
6. Apply fix.
7. Validate recovery.
8. Document root cause.
9. Add future prevention steps.

## Example Incident

If the website does not load, the team checks whether the VM is running, whether Apache is active, and whether firewall rules allow HTTP.
