# Risks and Mitigations

## Cost Risk

Cloud resources may create charges.

Mitigation: Stop or delete resources after screenshot evidence is collected.

## Security Risk

Public SSH access can be dangerous.

Mitigation: Restrict SSH where possible and document hardening recommendations.

## Connectivity Risk

SSH or web access may fail.

Mitigation: Check public IP, firewall rules, VM status, and Apache status.

## Documentation Risk

A project without documentation may look like a basic lab.

Mitigation: Include architecture, deployment guides, runbooks, monitoring, and security documentation.
