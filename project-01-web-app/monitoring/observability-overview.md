# Observability Overview

## Purpose

Observability helps cloud engineers understand what is happening inside systems by using metrics, logs, dashboards, and alerts.

## Project Observability Layers

1. Metrics from cloud monitoring platforms.
2. Logs from platform logging tools.
3. Health checks using browser validation and curl.
4. Runbooks for operational response.
5. Troubleshooting documentation.

## Observability Outcome

The project demonstrates basic observability across AWS, Azure, and GCP.
