# Service Level Objectives

## Purpose

Service Level Objectives define reliability goals for the application.

## Demo Environment SLO

For this portfolio project, the target availability during active testing is:

- 99% availability during demo and validation windows

## Why This SLO Makes Sense

This is not a production customer-facing service, but defining an SLO demonstrates SRE thinking. The goal is to ensure the application is reachable when being tested, reviewed, or demonstrated.

## Future SLOs

A production version could define:

- 99.9% monthly availability
- Page response under 500ms
- Recovery within 30 minutes
