# Security Policy - Multi-Cloud Incident Response Lab

## Purpose

This policy defines how security controls should be configured, reviewed, and documented for this project.

## Standards

- Use least privilege access.
- Enable audit logging.
- Require MFA where supported.
- Do not store secrets in code.
- Capture screenshots as evidence.
- Review access and policies regularly.

## Evidence Required

- Configuration screenshot.
- Log or finding screenshot.
- Access/policy screenshot.
- Architecture diagram.
- GitHub repository screenshot.
